# API REST GitHub — L’essentiel

L’API REST de GitHub permet d’automatiser, d’intégrer et d’étendre les fonctionnalités de GitHub via des scripts ou applications.

**Cas d’usage principaux :**
- Trier et gérer les issues
- Créer des tableaux de bord d’analyse
- Gérer les versions et releases
- Automatiser des workflows

**Ressources utiles :**
- [Démarrage rapide de l’API REST GitHub](https://docs.github.com/fr/rest/quickstart?apiVersion=2022-11-28)
- [Authentification à l’API REST](https://docs.github.com/fr/rest/overview/authenticating-to-the-rest-api?apiVersion=2022-11-28)
- [Liste des endpoints REST](https://docs.github.com/fr/rest/reference?apiVersion=2022-11-28)

**Bonnes pratiques :**
- Sécuriser ses identifiants
- Respecter les limites de taux
- Utiliser les webhooks pour recevoir des notifications
